## 1. 출발점

초기 질문은 단순한 압축 문제가 아니었다.

> 복잡한 모델을 잘 압축한다는 것은 정보를 무작정 버리는 일이 아니라, 그 대상에 맞는 구조적 prior와 좌표계를 찾는 일인가?

이 직관은 신호 처리와 표현 학습의 고전적 예시들에서 출발했다.

- `JPEG / DCT`: 자연 영상은 저주파에 에너지가 몰리므로 적절한 좌표계에서 압축이 쉬워진다.
- `Wavelet`: edge가 많은 신호는 DCT보다 국소적 공간-주파수 표현이 유리하다.
- `PCA / Eigenfaces`: 고차원 얼굴 데이터도 저차원 구조 위에 놓일 수 있다.
- `Sparse coding`: 자연 영상 patch는 학습된 dictionary 위의 희소 조합으로 설명될 수 있다.
- `LPC / CELP`: 음성은 생성 모델과 잔차의 조합으로 다루는 것이 효과적이다.
- `MPEG`: 비디오는 전체를 한 번에 다루지 않고 motion model과 residual로 나눈다.

핵심 교훈은 하나다.

> 좋은 압축은 하나의 만능 기저를 찾는 일이 아니라, 대상에 맞는 구조적 prior를 찾는 일이다.

## 2. 이 관점을 LLM에 적용하면

초기에는 다음과 같은 강한 가설이 있었다.

> LLM 전체에도 숨겨진 수학 구조가 있고, 올바른 좌표계를 찾으면 더 간결하게 표현할 수 있지 않을까?

하지만 논의가 진행되며 이 framing은 너무 강하다고 판단됐다.

- LLM 전체를 하나의 수학식이나 하나의 좌표계로 설명하려는 시도는 범위가 너무 크다.
- “hidden formula” 같은 표현은 검증 불가능한 형이상학으로 오해받기 쉽다.
- 실제로는 모델 전체보다 컴포넌트별 구조가 더 현실적인 분석 단위일 수 있다.

그래서 방향이 이렇게 바뀌었다.

### 기존 framing

- 모델 전체를 하나의 압축 대상으로 본다.
- 하나의 전역 manifold 또는 좌표계를 찾으려 한다.
- compression이 연구의 중심이다.

### 발전된 framing

- 모델은 heterogeneous component의 조합이다.
- 각 component는 서로 다른 structural prior를 가질 수 있다.
- 먼저 adaptation geometry를 측정하고, 그 결과를 바탕으로 architecture를 논의한다.

즉, `compression first`에서 `geometry first`로 무게중심이 이동했다.

## 3. 컴포넌트별 prior라는 생각

논의 중 가장 오래 살아남은 아이디어는 “component-aware prior”이다.

LLM의 각 부분은 서로 다른 방식으로 구조화될 수 있다.

### Positional Encoding

- 이미 closed-form에 가깝다.
- 명시적 수학 구조가 가장 분명한 부분이다.

### Attention Projections

- low-rank 구조를 가질 가능성이 높다.
- LoRA류의 성공과도 자연스럽게 연결된다.

### Attention Heads

- circuit 또는 motif 수준의 기능적 구조를 가질 수 있다.
- 단순 행렬이 아니라 작은 알고리즘 단위로 볼 수 있다.

### FFN Blocks

- sparse dictionary 또는 overcomplete feature basis 관점이 유력하다.
- SAE 연구와 가장 잘 맞물리는 영역이다.

### Residual Stream

- 여러 컴포넌트를 이어주는 shared feature space일 가능성이 있다.
- 동시에 component coupling이 가장 크게 드러나는 지점이기도 하다.

이 관점은 이후 `future-directions.md`로 정리된 장기 비전의 핵심 토대가 되었다.

## 4. TinyLoRA가 촉발한 전환

논의의 큰 전환점은 TinyLoRA류 결과였다.

작은 수의 파라미터만 학습해도 reasoning task에서 높은 성능이 나오는 현상은 다음 질문을 강하게 만들었다.

> 이 작은 update는 정말로 새로운 지식을 “주입”하는 것인가, 아니면 이미 있는 capability를 “선택/활성화”하는 것인가?

이 질문에서 나온 중심 가설이 나중의 ASCENT 중심 문장이 된다.

> Task adaptation in pretrained LLMs is primarily a control problem over a pretrained capability substrate, not a knowledge injection problem.

여기서 중요한 변화는 다음과 같다.

- adaptation을 knowledge injection이 아니라 control 문제로 본다.
- pretrained model 내부에는 이미 latent capability substrate가 있다고 본다.
- 작은 update의 역할은 그 substrate 위에서 올바른 방향을 선택하는 것이라고 본다.

## 5. Capability Library에서 Capability Substrate로

중간 단계에서는 모델을 “capability library”로 보는 표현이 자주 등장했다.

즉, pretrained model 안에는 여러 기능 조각이 저장돼 있고, task adaptation은 그 라이브러리에서 필요한 기능을 선택하는 과정이라는 관점이다.

하지만 이 표현도 곧 한계가 드러났다.

- library라는 말은 너무 정적인 느낌을 준다.
- discrete module이 있다는 인상을 강하게 주지만, 실제 내부 구조는 superposition을 가질 수 있다.
- 연속적인 feature space나 overlapping structure를 설명하기 어렵다.

그래서 더 유연한 표현으로 `capability substrate`가 채택됐다.

이 표현은 다음을 허용한다.

- 기능이 명확히 분리된 모듈일 수도 있고,
- 겹쳐진 feature 조합일 수도 있으며,
- 계층적 timescale 구조를 가질 수도 있다.

## 6. 장기 아키텍처 비전

원본 노트에는 첫 논문의 범위를 넘어서는 장기 비전도 강하게 등장한다.

### 6.1 Compact Dynamics Core

모델의 “지능”은 큰 dense stack 전체에 퍼져 있다기보다, 작은 동역학 코어에 집중될 수 있다는 생각이다.

핵심 형태는 다음과 같이 요약된다.

```text
z_{t+1} = G(z_t, x_t, m_t)
```

- `z_t`: 작은 latent state
- `G`: 반복 적용되는 update rule
- `m_t`: memory readout

### 6.2 Memory Separation

모든 지식을 weight 안에 억지로 넣는 대신:

- control / reasoning
- stored knowledge
- retrieval

를 분리할 수 있다는 아이디어가 제안됐다.

### 6.3 Space-Time Dynamics

모델의 계산은 depth와 time의 두 축으로 나눠볼 수 있다는 관점이 등장했다.

- depth axis: layer 간 반복 구조
- time axis: token 단위 selective dynamics

이 관점은 나중에 framework 문서에서 `space-time dynamics`로 정리되었다.

### 6.4 Self-Similarity와 Hierarchy

초기에는 “LLM은 fractal이다”에 가까운 강한 표현도 있었지만, 최종적으로는 다음처럼 정리됐다.

- self-similarity는 측정 대상이다.
- hierarchy는 표현 구조의 성질이다.
- fractal은 비유나 ontological claim이 아니라 분석 언어로 제한해야 한다.

## 7. 연구 전략의 대전환: Architecture First에서 Geometry First로

가장 중요한 전략 변화는 여기서 일어났다.

### 버려진 접근

- 새 아키텍처를 먼저 제안한다.
- 그 아키텍처를 정당화할 이론을 나중에 붙인다.
- 크고 멋진 비전을 앞세운다.

### 채택된 접근

- 먼저 adaptation geometry를 측정한다.
- shared subspace가 있는지 본다.
- task direction이 separable한지 본다.
- cross-model transfer가 되는지 본다.
- 결과가 좋을 때만 architecture 논의를 뒤에 붙인다.

이때 정리된 판단은 매우 명확하다.

> 첫 번째 논문의 싸움은 “새 아키텍처를 믿게 하는 것”이 아니라, “tiny adaptation이 정말 구조적인 control problem인지”를 보여주는 것이다.

## 8. 실험으로 내려온 핵심 가설

이 전환의 결과, 원래 넓고 철학적이던 아이디어는 몇 개의 실험 가설로 압축되었다.

### H1a. Shared Adaptive Subspace

- 여러 task의 update vector가 공통 저차원 subspace에 놓이는가?

### H1b. Directional Separability

- 그 공통 subspace 안에서도 task별 방향이 서로 구분되는가?

### H2. Transferability

- 한 모델에서 얻은 task update가 유사한 다른 모델로 transfer되는가?

### H1c / H3

- semantic alignment
- self-similarity

는 남겨두되 exploratory 항목으로 내렸다.

즉, 첫 논문의 evidential burden은 `H1a`, `H1b`, `H2`가 맡는다.

## 9. Pre-registration으로 정제된 이유

원본 노트 후반부의 큰 비중은 pre-registration 문서를 깐깐하게 다듬는 데 쓰였다.

왜냐하면 이 프로젝트의 가장 큰 위험은 다음이었기 때문이다.

- 실험 후에 결과를 해석하는 방향으로 claim을 바꾸는 것
- 애매한 threshold를 남겨두고 post-hoc reinterpretation 하는 것
- exploratory 결과를 central claim처럼 사용하는 것

그래서 다음 항목이 문서에 고정되었다.

- H1a의 gray zone
- H1b의 circularity 제거
- update vector 정의
- normalization과 magnitude 분석 분리
- RL 기반 primary analysis와 SFT baseline 분리
- stopping rule
- falsification rule
- recovery path

그 결과, 이 프로젝트는 단순 아이디어 메모가 아니라 “등록 가능한 empirical study”로 바뀌었다.

## 11. 버려진 주장과 남겨진 주장

### 버려진 주장

- “LLM 전체에 hidden formula가 있다”
- “LLM은 fractal이다”
- “새 아키텍처를 지금 바로 제안해야 한다”
- “H3 같은 exploratory 결과가 central claim을 좌우해야 한다”

### 남겨진 주장

- adaptation은 control 문제일 수 있다
- pretrained model 내부에는 reusable substrate가 있을 수 있다
- 그 구조는 update geometry를 통해 측정 가능할 수 있다
- architecture는 geometry 결과 다음 단계에서 논의해야 한다
