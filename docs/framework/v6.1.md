## 1. Central Claim

> **Task adaptation in pretrained LLMs is primarily a control problem over a pretrained capability substrate, not a knowledge injection problem.**

This single sentence is the falsifiable core of the ASCENT framework. Everything else is either supporting structure, measurement methodology, or architectural implication.

### What this claim says

- Task-specific updates activate existing capabilities, rather than creating new ones
- The dominant cost of adaptation is **control** (selecting/activating the right subspace), not **injection** (storing new information)
- This implies a structural property of the pretrained substrate: it must already contain latent capabilities addressable by small updates

### What this claim does not say

- It does not claim that all adaptation is purely control (some tasks may require genuine injection)
- It does not claim that the substrate has discrete, library-like units
- It does not claim this holds universally — it is **task-distribution relative** (see §3)

## 2. Supporting Structures

Four structures support the central claim without being part of it.

### 2.1 Task-Relative Information Decomposition

```
B_model[T] = B_shared[T] + B_adaptive[T]
```

- `T` = pre-specified task distribution
- `B_shared[T]` = mutual information between model and features shared across T
- `B_adaptive[T]` = additional information needed to specify a particular task within T

**Key observation:** TinyLoRA update size is a proxy for `B_adaptive[T_reasoning]`.  
The paper shows this is approximately 13 parameters ≈ 26 bytes for T_reasoning.

**Important:** This decomposition is **not absolute**. It is defined relative to T.  
Different T gives different B_shared, B_adaptive. No universal shared information is claimed.

### 2.2 Substrate Hierarchy

The capability substrate is stratified by update timescale:

```
S = { S^fast, S^slow, S^base, S^meta }
```

| Layer | Timescale | Updated By | Mechanism |
|-------|-----------|------------|-----------|
| `S^fast` | Per-token, per-sequence | In-context dynamics | KV cache, attention |
| `S^slow` | Per-task, per-session | Fine-tuning, LoRA | TinyLoRA adapter |
| `S^base` | Per-model | Pre-training | Base LLM weights |
| `S^meta` | Per-design | Architecture choice | Layer structure, width |

**Implication:** In-context learning ≈ `S^fast` modulation.  
TinyLoRA / RL adaptation ≈ `S^slow` control.  
ASCENT-G experiments primarily investigate `S^slow` structure.

### 2.3 Space-Time Dynamics

The framework separates two axes of computation:

**Depth axis** (layers):
```
z^(ℓ+1, t) = F_θ(z^(ℓ,t); γ_ℓ, c^slow)
```
- `F_θ`: shared core operation (self-similar across layers)
- `γ_ℓ`: hierarchical per-layer modulation
- `c^slow`: task-level controller (slow, stable)

**Time axis** (tokens):
```
z^(ℓ, t+1) = G_φ(z^(ℓ,t), x_t, m^(ℓ,t), c^fast_t)
```
- `G_φ`: input-driven selective dynamics
- `m^(ℓ,t)`: capability memory readout
- `c^fast_t`: token-level controller (fast, dynamic)

**Memory access:**
```
m^(ℓ, t) = Read(q^(ℓ,t), L^(ℓ))
q^(ℓ,t)  = Q[z^(ℓ,t), c^fast_t, c^slow]
```

**Note:** This is a conceptual framework for ASCENT-A (Paper 2), not an architectural proposal in Paper 1.

### 2.4 Ontology Ladder

Experimental results determine how strong a claim can be made:

| Result | What Can Be Claimed |
|--------|---------------------|
| H1a fail | No shared adaptive structure |
| H1a inconclusive | Weak evidence only |
| H1a marginal pass | Possible shared update manifold |
| H1a strong pass + H1b fail | Shared subspace, non-separable |
| H1a strong pass + H1b pass | Capability-like decomposition |
| All above + H1c pass | Interpretable capability substrate |

**Rule:** Never claim a higher ontological level than the experimental evidence supports.

## 3. Task-Relativity Principle

All claims in ASCENT are **conditioned on a task distribution T**.

```
"Adaptation is primarily control" ← claim about T_reasoning
                                  ← may not hold for T_code, T_language
```

Pre-specified distributions:

```
T_reasoning = {GSM8K, MATH, AIME, AMC, MATH500,
               HumanEval, MBPP, CommonsenseQA,
               HellaSwag, ARC-Challenge}

T_language  = {Translation, Summarization, ...}   [future work]
T_code      = {HumanEval, MBPP, ...}               [overlap with T_reasoning]
```

ASCENT-G tests claims about `T_reasoning` only. Generalization to other T is future work.

## 4. Controller Feasibility Principle

A design principle (not a theorem):

> A small-controller architecture is only plausible if the effective number of task-useful selectable subspaces grows slowly enough to be addressable by the controller.

Empirical proxy: if H1a passes with r_90 small relative to D, then the number of distinct useful directions is tractable for a small controller.

This principle motivates the architecture in ASCENT-A, pending evidence from ASCENT-G.

## 5. Relationship to Prior Work

### 5.1 What ASCENT adds

Existing work (QuaRot, SpinQuant, TurboQuant) treats adaptation as coordinate-space transformation. ASCENT reframes:

```
Old frame: "Find better coordinates for compression"
New frame: "Understand the control structure of the substrate"
```

ASCENT adds:
1. Information-theoretic decomposition B_shared / B_adaptive
2. Explicit substrate hierarchy with timescales
3. Falsifiable measurement via update vector geometry
4. Architectural implications (ASCENT-A)

### 5.2 What prior work establishes (Evidence base)

| Work | What it establishes for ASCENT |
|------|-------------------------------|
| TinyLoRA | B_adaptive is small for T_reasoning |
| Universal Transformer | F_θ can be shared across depth |
| ALBERT | Cross-layer sharing is practical |
| Mamba | G_φ selective dynamics work |
| Anthropic SAE | S^base has interpretable structure |
| Ansuini et al. | Representation intrinsic dim < model dim |
| Scaling Laws (Kaplan) | Self-similar structure across scales |

## 7. Falsification Rules

### Hard falsification

Central claim is **strongly disfavored** if both:
- H1a fails AND H2 fails

In this case: reframe as mechanistic study of TinyLoRA without substrate hypothesis.

### Soft falsification

| Outcome | Consequence |
|---------|-------------|
| H1a pass, H1b fail | Claim "shared update manifold" only |
| H1b pass, H1c fail | No interpretability claim |
| H3 fail | No effect on central claim (H3 is exploratory) |

### Recovery path

If hard falsification occurs, alternative paper direction:

> *"Understanding TinyLoRA Without Structured Adaptive Subspaces:  
> Decision Boundary Smoothing, Implicit Regularization, or Output Distribution Shift?"*

Negative results are publishable and valuable.

## 9. Open Questions

These are the genuinely unknown quantities that experiments will answer:

1. **Is H1a true?** Does shared subspace exist in T_reasoning?
2. **What is the effective dimensionality?** How large is r_90 / min(N, √D)?
3. **Is it interpretable?** Does H1c pass (semantic alignment)?
4. **Does it transfer?** Does H2 pass across model variants?
5. **Is S^base self-similar?** Does H3 provide evidence for shared F_θ?
6. **What is B_adaptive for other T?** Does the pattern generalize?

ASCENT-G answers questions 1-5. Questions 6+ are future papers.
